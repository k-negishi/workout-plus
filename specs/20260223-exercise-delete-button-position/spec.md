# 仕様書: 種目削除ボタンの配置修正（Issue #137）

## 概要

登録画面（RecordScreen）の種目ブロック（ExerciseBlock）において、種目削除ボタン（✕）の位置を変更する。
現在は「前回記録バッジの左側」にあるが、「前回記録バッジの右側」に移動する。

## 背景・目的

- GitHub Issue: https://github.com/k-negishi/workout-plus/issues/137
- ユーザーが前回記録バッジを操作しようとした際に誤タップが発生しやすいため、削除ボタンをバッジの右端に移動する
- バッジを先に視認した上でその右に削除ボタンがある方が、UI の自然な読み順（左→右）と一致する

## 現状

`ExerciseBlock.tsx` のヘッダー右エリアのレイアウト:

```
[種目名 / 部位ラベル]    [✕] [前回記録バッジ]
```

## 変更後

```
[種目名 / 部位ラベル]    [前回記録バッジ] [✕]
```

## 機能要件

### FR-1: ボタン順序の変更
- **変更前**: `[✕ボタン] [前回記録バッジ]`
- **変更後**: `[前回記録バッジ] [✕ボタン]`
- ✕ボタンは前回記録バッジが非表示のときも右端に表示される

### FR-2: 外観の維持
- ✕ボタンのスタイル（サイズ・色・フォント・padding）は変更しない
- 前回記録バッジのスタイルも変更しない
- `gap: 8` の間隔も維持する

### FR-3: アクセシビリティ
- `accessibilityLabel` は変更しない

## 非機能要件

### NFR-1: テスト
- 既存の `ExerciseBlock.test.tsx` のテストが引き続きパスすること
- ボタンの順序変更を検証するテストを追加すること

### NFR-2: 型・Lint
- TypeScript のエラーなし
- ESLint・Prettier エラーなし

## 影響範囲

| ファイル | 変更内容 |
|---|---|
| `apps/mobile/src/features/workout/components/ExerciseBlock.tsx` | ヘッダー右エリアの要素順序を変更 |
| `apps/mobile/src/features/workout/components/__tests__/ExerciseBlock.test.tsx` | 順序検証テストを追加 |

## 対象外

- SetRow の削除ボタンは変更しない（セット行の ✕ は別物）
- 削除確認ダイアログの有無は変更しない
